# DeepSeek 交易机器人统一版本 PRD

## 产品愿景

将现有四个版本的 DeepSeek 交易机器人整合为一个统一、模块化、易维护的生产级系统。结合各版本优势，创建一个功能完善、性能优异、可扩展的 BTC/USDT 永续合约交易机器人。

## 产品背景

当前项目拥有多个版本：
- **v1.0**: 基础 Binance 版本 (396行)
- **v2.0**: OKX 基础版本 (414行)
- **v3.0**: 技术指标增强版 (730行)
- **v4.0**: 完整策略版 (1111行)

各版本功能互补但代码重复严重，需要统一整合。

## 功能需求 (FR)

### FR-1: 多交易所支持
**优先级**: P0
**描述**: 系统应支持 Binance 和 OKX 两个主要交易所

**验收标准**:
- ✅ 支持 Binance 现货/期货交易
- ✅ 支持 OKX 永续合约交易
- ✅ 交易所配置通过环境变量管理
- ✅ 自动检测和切换交易所类型
- ✅ 统一的交易接口抽象层

**依赖**: 技术指标计算模块

---

### FR-2: 技术指标分析引擎
**优先级**: P0
**描述**: 提供完整的技术分析能力

**验收标准**:
- ✅ 移动平均线 (SMA/EMA): 5/20/50/96周期
- ✅ MACD 指标: DIF, DEA, MACD柱
- ✅ RSI 相对强弱指标
- ✅ 布林带: 上轨、中轨、下轨、位置百分比
- ✅ 支撑阻力位计算
- ✅ 市场趋势分析
- ✅ 成交量分析
- ✅ 指标参数可配置

**技术细节**:
- 使用 Pandas 进行数据处理
- 支持实时计算和历史数据回测
- 指标计算结果缓存机制

---

### FR-3: DeepSeek AI 分析引擎
**优先级**: P0
**描述**: 集成 DeepSeek AI 进行市场分析和信号生成

**验收标准**:
- ✅ DeepSeek Chat API 集成
- ✅ 结构化提示词设计
- ✅ JSON 格式响应解析
- ✅ 错误处理和重试机制
- ✅ 备用信号生成逻辑
- ✅ 多维度分析 (技术+情绪+趋势)

**分析维度**:
1. K线形态分析
2. 技术指标综合评估
3. 支撑阻力位判断
4. 市场情绪数据 (可选)
5. 趋势连续性分析

---

### FR-4: 智能仓位管理
**优先级**: P0
**描述**: 根据信号信心度和市场趋势动态调整仓位

**验收标准**:

- ✅ 信心度分级仓位:
  - HIGH: 1.5x 基础仓位

- ✅ 全仓模式优先
- ✅ 动态风险控制

- 技术指标计算 < 500ms
- 订单执行响应 < 2秒

**验收标准**:
- ✅ 15分钟周期内完成所有分析和交易

## 约束条件

### 技术约束
- **语言**: Python 3.10+
- **主要依赖**: CCXT, Pandas, OpenAI, Schedule, python-dotenv
- **交易所**: Binance, OKX
- **AI 服务**: DeepSeek Chat API
- **平台**: Linux (Ubuntu 20.04+ 推荐)

### 业务约束
- **交易对**: BTC/USDT (可扩展)
- **杠杆范围**: 1x - 100x
- **最小仓位**: 根据交易所规格
- **交易频率**: 最小间隔 15 分钟
- **持仓模式**: 单向持仓 + 全仓模式

### 安全约束
- **测试模式**: 首次部署必须启用
- **API 权限**: 仅使用交易必需的权限
- **密钥管理**: 禁止硬编码密钥

## 风险与缓解

| 风险项 | 影响 | 概率 | 缓解措施 |
|--------|------|------|----------|
| **AI 分析错误** | 高交易损失 | 中 | 多维度验证、保守策略、严格止损 |
| **API 限流** | 服务中断 | 中 | 指数退避重试、请求频率控制 |
| **网络延迟** | 订单执行失败 | 中 | 超时设置、异常重试 |
| **市场异常波动** | 大额亏损 | 低 | 智能仓位管理、��大持仓限制 |
| **代码缺陷** | 系统故障 | 低 | 测试模式、异常处理、监控告警 |
